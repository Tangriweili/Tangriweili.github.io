安装 glibc-all-in-one

一个专注于 管理多版本 glibc 环境 的工具集，用于快速下载、配置和切换不同版本的 glibc 动态库（libc.so.6）和链接器（ld.so）。适用于：

  • ✅ CTF Pwn 题目调试（尤其堆题需特定 libc 版本）148

  • ✅ 漏洞复现（如堆漏洞利用依赖低版本 glibc）36

  1. 下载 glibc 版本

    • 通过 patchelf 修改二进制文件，强制其使用指定版本的 ld 和 libc136。

LibcSearcher安装

以下是 LibcSearcher 的详细介绍及基本用法指南，专为 CTF Pwn 设计：

🔧 LibcSearcher 是什么？

一个 Python 库，用于自动化匹配 libc 版本并计算符号偏移。通过泄露的函数地址（如 puts、printf），自动在本地/云端数据库查找匹配的 libc 版本，获取关键符号（system, execve, /bin/sh）的偏移地址。

核心功能：
✅ 自动匹配 - 基于泄露的函数地址识别 libc 版本
✅ 偏移计算 - 一键获取 system、/bin/sh 等关键符号偏移
✅ 多条件筛选 - 支持多个泄露地址提高匹配精度

我遇到个问题我明明改了~/.bashrc文件，重新通过ssh登录的时候每次我都要手动输入source ~/.bashrc，配置才会生效，很是头疼，于是我就研究了一下解决办法以及问题的原因是什么。
先说解决的方法，在家目录下的.bash_profile 文件中添加source ~/.bashrc 的信息：

使用vim 打开

vim ~/.bash_profile
if test -f ~/.bashrc ; then
source ~/.bashrc
fi

如果目录下没有，vim 也会自动生成。
接着，每次登陆shell 后它就会自动运行bashrc了。
